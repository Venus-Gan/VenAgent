# 调研 (Research)

本目录存放项目启动前的技术调研、审计报告和技术选型评估。

## 文件清单

| 文件 | 说明 |
|------|------|

## 维护约定

- 基线型调研文档应标注审计日期；当实现事实显著变化时，更新当前审计或将旧版移入 `archive/`，不得让过时结论继续充当当前事实
- 新的专项技术评估以独立文件追加
- 备选技术方案的详细对比放在 `technology-eval/` 子目录下

## 阅读建议

- [technical-audit.md](./technical-audit.md) 是项目重构的**事实起点**，建议在阅读任何 SDD 之前先浏览
- 它回答了"我们现在有什么、问题在哪里、应该先修什么"，是后续 PRD 和 SDD 的输入
