# ADR-011: 冻结 AGI-saber 行为参考版本

## 状态

已接受

## 背景

AGI-saber 可以帮助解释部分历史行为，但持续同步会把外部代码、依赖和安全范围引入 VenAgent，造成目标架构漂移和不可审计的变更来源。

## 备选方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| 持续同步外部仓库 | 可获得上游更新 | 来源漂移、依赖扩大、审查边界不清 |
| 完全不参考外部行为 | 范围最小 | 可能丢失兼容性线索 |
| 冻结一个参考提交，仅静态比对 | 可复现、可审计，同时保留行为线索 | 后续差异需要人工记录和迁移 |

## 结论

AGI-saber 仅作为冻结的静态行为参考，不属于 VenAgent 的运行、部署或持续同步路径。参考提交为
`python@2b995cdd8b2fb413bfb34c41456ec0bda92e6c2a`；任何移植都必须进入 VenAgent 的测试、代码审查和安全审查。

## 后果

- 不执行外部账号、部署、推送或自动同步操作。
- 参考仓库中的凭据和历史问题不自动扩大为 VenAgent 的运行面；VenAgent 仍须独立扫描自身工作树和历史。
- 后续若改变范围，必须新增 ADR 或显式修订本 ADR。
