# ADR (Architecture Decision Records)

架构决策记录。记录那些"一旦定下来就很难回头"的关键决策。

## 格式规范

每个 ADR 文件包含以下段落：

```markdown
# ADR-001: 选择 LangGraph 作为编排引擎

## 状态
提议 / 已接受 / 已废弃（被 ADR-XXX 取代）

## 背景
当前状态是什么？为什么需要做这个决策？

## 备选方案
| 方案 | 优点 | 缺点 |
|------|------|------|
| A    | ...  | ...  |
| B    | ...  | ...  |

## 结论
选择了哪个方案？理由是什么？

## 后果
这个决策会带来什么正面/负面影响？需要做什么后续工作？
```

## 编号规则

- 按时间顺序递增：`001-`、`002-`、`003-`…
- 标题使用英文短 slug，如 `001-adopt-langgraph.md`

## 维护约定

- ADR **只追加，不覆盖**——即使后来的决策推翻了之前的，也只追加新 ADR 并引用旧 ADR（状态标记为"已废弃"）
- 不要在 SDD 中重复记录已写入 ADR 的决策细节，SDD 中引用 ADR 编号即可
