# RuntimeGovernance 方案

## 定位
| 项目 | 约定 |
|---|---|
| RuntimeGovernance | 运行时控制面，统一管理任务如何安全运行 |
| 负责 | 生命周期、执行策略、状态可见性、恢复与快照 |
| 不负责 | RAG / memory / promptctx / agentteam / MCP 的具体业务实现 |

## 负责范围
| 能力 | 说明 |
|---|---|
| 生命周期控制 | 启动、运行、暂停、中断、恢复、结束、清理 |
| 执行态治理 | 超时、重试上限、并发上限、取消语义、失败策略 |
| 状态可见性 | 当前阶段、已完成节点、失败节点、是否取消、最近快照 |
| 恢复与快照 | 业务快照、图执行 checkpoint、恢复点选择、恢复策略 |

## 推荐组件
| 组件 | 作用 |
|---|---|
| checkpoint | 保存图执行态 |
| restore | 恢复执行态和业务上下文 |
| cancel | 停止运行中的任务 |
| status | 聚合运行态并对外展示 |
| policy | 定义超时、重试、并发和失败语义 |

## 与现有代码对应
| 现有模块 | 归属 |
|---|---|
| restore.py | 启动恢复与知识图谱挂载 |
| snapshot.py | 业务快照持久化 |
| cancel.py | 取消注册表与中断语义 |
| status.py | 运行态聚合与对外状态视图 |

## 与 LangGraph 的边界
| 层 | 负责 |
|---|---|
| LangGraph | 图级执行与状态流转 |
| RuntimeGovernance | 任务级 / 应用级控制策略 |
| checkpoint | 保存执行态 |
| restore | 恢复执行态 |
| cancel | 停止执行态 |

## 验收标准
- 可以统一查看任务当前状态
- 可以统一选择恢复点
- 可以统一处理取消和超时
- 业务快照与执行 checkpoint 分开管理
